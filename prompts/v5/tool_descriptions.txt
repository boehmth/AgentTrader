Verfügbare Tools (harmonisiertes Interface):

Alle Tools nehmen genau drei String-Argumente entgegen:
  operation, operand1, operand2

Zahlen werden IMMER als Strings übergeben (z. B. "3.14", "100").
Nicht benötigte Argumente werden als leerer String "" übergeben.

============================================================
TOOL: get_prices
============================================================
Zweck:
  Liefert die Historie der Schlusskurse für einen Ticker.
  Für dieses Setup: nutze IMMER operand2="20" (20-Tage-Historie).
  Damit hast du für jede Kennzahl (5-Tage, 20-Tage) genug Daten.

Args:
  { "operation": "get",
    "operand1": "<ticker: NVDA | MSFT | SAP | AMZN>",
    "operand2": "20" }

Ergebnis:
  {
    "symbol": "<ticker>",
    "prices": [
      { "date": "YYYY-MM-DD", "close": <float> },
      ...  20 Einträge, aufsteigend nach Datum
    ],
    "latest": { "date": "YYYY-MM-DD", "close": <float> }
  }

Hinweise zum Zugriff über $results-Referenzen:
  - Aktueller Preis (latest close):
      "$results[i].latest.close"
  - Preis vor 20 Handelstagen:
      "$results[i].prices.0.close"
  - Preis vor 5 Handelstagen (Index 15 = 20-5):
      "$results[i].prices.15.close"

============================================================
TOOL: calculator
============================================================
Zweck:
  Führt eine exakte arithmetische Operation aus.
  Rechne Beträge, Renditen etc. NIE selbst — immer über den Calculator.

Args:
  { "operation": "add | subtract | multiply | divide",
    "operand1": "<number as string>",
    "operand2": "<number as string>" }

Ergebnis:
  { "result": <float> }

Standard-Berechnungen (Rezepte):

  20-Tage-Rendite in %:
    step A: subtract(latest.close, prices.0.close)     -> diff
    step B: divide(diff, prices.0.close)               -> ratio
    step C: multiply(ratio, 100)                       -> % Rendite

  5-Tage-Rendite in %:  wie oben, aber mit prices.15.close statt prices.0.close.

  Kaufwert:
    multiply(shares, price)                            -> cost

  Anteil einer Position am Gesamtwert:
    step A: multiply(shares, latest.close)             -> position_value
    step B: add(cash, sum_aller_position_values)       -> total
    step C: divide(position_value, total)              -> share_ratio
    step D: multiply(share_ratio, 100)                 -> % Anteil

============================================================
TOOL: portfolio
============================================================
Zweck:
  Verwaltet Cash und Aktienbestände.
  Der initiale Cash-Bestand ist bereits gesetzt — du kannst
  ihn NICHT ändern. Du kannst nur handeln (kaufen/verkaufen).

Operationen:
  - load : liefert aktuellen Zustand (Cash + Holdings).
  - buy  : kauft N Aktien zu Preis P.
  - sell : verkauft N Aktien zu Preis P.

Args:
  load:
    { "operation": "load", "operand1": "",         "operand2": "" }
  buy:
    { "operation": "buy",  "operand1": "<ticker>", "operand2": "<shares>@<price>" }
  sell:
    { "operation": "sell", "operand1": "<ticker>", "operand2": "<shares>@<price>" }

Beispiele für operand2:
  "2@145.30"    -> 2 Aktien zu 145.30
  "0.5@980.00"  -> 0.5 Aktien zu 980.00

WICHTIG:
  - Nutze IMMER den aktuellen "latest.close" als Preis in operand2:
      "operand2": "10@$results[i].latest.close"
    NICHT einen alten Preis, NICHT einen ausgedachten Preis.
  - "shares" sollte eine ganze Zahl sein (z. B. "12", nicht "12.37"),
    außer du hast einen sehr guten Grund für Bruchteile.

Ergebnis (Erfolg):
  {
    "date": "YYYY-MM-DD",
    "cash": <float>,
    "holdings": { "NVDA": <float>, "MSFT": <float>, "SAP": <float>, "AMZN": <float> }
  }

Ergebnis (Fehler):
  { "error": "<message>" }

Wichtige Regeln:
- Du kannst nur kaufen, wenn du genug Cash hast.
- Du kannst nur verkaufen, was du hast.
- Cash < verfügbarem Cash bedeutet NICHT, dass das Portfolio leer ist —
  du hast dann einfach viele Aktien und wenig Cash.
