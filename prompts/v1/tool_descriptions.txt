Verfügbare Tools (harmonisiertes Interface):

Alle Tools nehmen genau drei String-Argumente entgegen:
  operation, operand1, operand2

Zahlen werden IMMER als Strings übergeben (z. B. "3.14", "100").
Nicht benötigte Argumente werden als leerer String "" übergeben.

============================================================
TOOL: get_prices
============================================================
Zweck:
  Liefert die Historie der Schlusskurse für einen Ticker.
  Der LLM entscheidet über das Zeitfenster (Anzahl Tage).

Args:
  { "operation": "get",
    "operand1": "<ticker: NVDA | MSFT | SAP | AMZN>",
    "operand2": "<days as string, default '1', max '365'>" }

Ergebnis:
  {
    "symbol": "<ticker>",
    "prices": [
      { "date": "YYYY-MM-DD", "close": <float> },
      ...
    ],
    "latest": { "date": "YYYY-MM-DD", "close": <float> }
  }

============================================================
TOOL: calculator
============================================================
Zweck:
  Führt eine exakte arithmetische Operation aus.
  Der LLM soll Beträge, Renditen etc. NIE selbst rechnen —
  immer den Calculator nutzen.

Args:
  { "operation": "add | subtract | multiply | divide",
    "operand1": "<number as string>",
    "operand2": "<number as string>" }

Ergebnis:
  { "result": <float> }

============================================================
TOOL: portfolio
============================================================
Zweck:
  Verwaltet Cash und Aktienbestände.
  Der initiale Cash-Bestand ist bereits gesetzt — du kannst
  ihn NICHT ändern. Du kannst nur handeln (kaufen/verkaufen).

Operationen:
  - load : liefert aktuellen Zustand (Cash + Holdings).
  - buy  : kauft N Aktien zu Preis P.
  - sell : verkauft N Aktien zu Preis P.

Args:
  load:
    { "operation": "load", "operand1": "",         "operand2": "" }
  buy:
    { "operation": "buy",  "operand1": "<ticker>", "operand2": "<shares>@<price>" }
  sell:
    { "operation": "sell", "operand1": "<ticker>", "operand2": "<shares>@<price>" }

Beispiele für operand2:
  "2@145.30"    → 2 Aktien zu 145.30
  "0.5@980.00"  → 0.5 Aktien zu 980.00

Ergebnis (Erfolg):
  {
    "date": "YYYY-MM-DD",
    "cash": <float>,
    "holdings": { "NVDA": <float>, "MSFT": <float>, "SAP": <float>, "AMZN": <float> }
  }

Ergebnis (Fehler):
  { "error": "<message>" }

Wichtige Regeln:
- Du kannst nur kaufen, wenn du genug Cash hast.
- Du kannst nur verkaufen, was du hast.
- Cash < verfügbarem Cash bedeutet NICHT, dass das Portfolio leer ist —
  du hast dann einfach viele Aktien und wenig Cash.

============================================================
WICHTIG
============================================================
- Es gibt NUR diese drei Tools.
- Es gibt NUR die oben aufgeführten Operationen.
- Es gibt NUR die Felder operation, operand1, operand2 in args.
